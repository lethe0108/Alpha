# 🧠 Memory Recall Skill

## 技能描述
基于钩子（hooks）的记忆检索系统，允许通过关键词触发快速回忆相关记忆。

## 触发条件
当用户询问关于身份、关系、事件、知识或配置的问题时，自动激活回忆技能。

## 核心功能

### 1. 钩子触发式回忆
- **关键词匹配**：自动识别用户问题中的关键词
- **钩子映射**：将关键词映射到相关记忆文件
- **智能排序**：按相关性排序回忆结果

### 2. 多维度记忆检索
- **身份记忆**：我是谁、我的能力、我的价值观
- **关系记忆**：南哥、婷姐等关系网络
- **事件记忆**：重要事件和时间线
- **知识记忆**：安全原则、网络配置、技能知识
- **配置记忆**：系统配置和设置

### 3. 上下文感知
- **对话历史**：考虑之前的对话内容
- **用户意图**：理解用户的真实需求
- **相关推荐**：推荐相关的记忆内容

## 使用方法

### 自动触发
当用户的问题包含以下关键词时自动触发：
- 谁、身份、我是谁
- 什么、怎么、为什么
- 昨天、今天、明天、时间
- 完成、任务、事件
- 安全、代理、配置
- 技能、能力、功能

### 手动调用
用户也可以直接说："回忆一下[关键词]" 或 "查找关于[主题]的记忆"

## 配置选项

### 记忆根目录
默认：`/root/.openclaw/workspace/memory/`

### 钩子匹配阈值
- **精确匹配**：1.0（完全匹配关键词）
- **高相似度**：>0.7
- **中相似度**：>0.5
- **低相似度**：>0.3

### 结果限制
- **最大结果数**：10
- **预览长度**：500字符
- **相关文件数**：3

## 技术实现

### 核心组件
1. **回忆引擎**：主处理逻辑，协调各个组件
2. **钩子匹配器**：智能匹配用户查询中的关键词
3. **记忆检索器**：高效检索记忆文件内容
4. **结果格式化**：将结果格式化为用户友好的响应

### 性能优化
- **缓存机制**：减少重复文件读取
- **并行处理**：同时搜索多个记忆类别
- **增量更新**：只更新变化的记忆文件

## 错误处理

### 常见错误
1. **文件不存在**：记忆文件被移动或删除
2. **权限问题**：无法读取记忆文件
3. **编码问题**：文件编码不兼容

### 恢复策略
1. **优雅降级**：返回部分结果或摘要
2. **错误报告**：记录错误并报告给用户
3. **自动修复**：尝试自动修复常见问题

## 安全考虑

### 隐私保护
- **敏感信息过滤**：过滤可能包含敏感信息的记忆
- **访问控制**：控制对敏感记忆的访问
- **审计日志**：记录所有记忆访问

### 数据完整性
- **文件验证**：验证记忆文件的完整性
- **备份机制**：定期备份记忆系统
- **版本控制**：支持记忆版本管理
